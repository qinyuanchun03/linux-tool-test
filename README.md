# Linux 模块化配置脚本
---

以下是针对我们改进的脚本编写的 `README` 更新部分，版本号为 `1.1.0`。更新内容涵盖了新增的功能、调用方式选择、动态加载子菜单等改进。

---

## **更新日志 - 版本 1.1.0**

### **新增功能**
1. **支持多种调用方式**：
   - 用户可以选择通过 **GitHub Raw**、**jsDelivr** 或 **Cloudflare** 调用脚本。
   - 调用方式选择菜单会在脚本启动时显示，用户可以根据需求选择最适合的方式。

2. **动态加载子菜单**：
   - 根据用户选择的调用方式，动态补全子菜单脚本的地址。
   - 子菜单脚本（如 `system_env.sh`、`network_env.sh` 等）会根据用户选择的调用方式加载，确保使用最新的内容。

3. **改进的用户体验**：
   - 提供了清晰的菜单选项和颜色提示。
   - 如果用户选择无效选项，会提示重新选择。

4. **代码优化**：
   - 新增了 `set_base_url` 和 `load_submenu` 函数，简化了代码逻辑。
   - 提高了脚本的可维护性和扩展性。

---

### **调用方式说明**
脚本支持以下三种调用方式：

1. **GitHub Raw**：
   - 直接调用 GitHub 的原始文件。
   - 示例命令：
     ```bash
     bash <(curl -sL https://github.com/qinyuanchun03/linux-tool-test/raw/main/index.sh)
     ```

2. **jsDelivr**：
   - 通过 jsDelivr 的 CDN 调用脚本，适合全球用户，尤其是中国大陆地区。
   - 示例命令：
     ```bash
     bash <(curl -sL https://cdn.jsdelivr.net/gh/qinyuanchun03/linux-tool-test@main/index.sh)
     ```

3. **Cloudflare**：
   - 通过 Cloudflare 的自定义域名调用脚本，适合需要更高安全性和自定义域名的场景。
   - 示例命令：
     ```bash
     bash <(curl -sL https://rules-key.jianghu.space/index.sh)
     ```

---

### **使用说明**
1. **运行脚本**：
   - 使用上述任意一种调用方式运行脚本。
   - 例如：
     ```bash
     bash <(curl -sL https://cdn.jsdelivr.net/gh/qinyuanchun03/linux-tool-test@main/index.sh)
     ```

2. **选择调用方式**：
   - 脚本启动后会显示调用方式选择菜单：
     ```
     +================================================+
     |                🌐 选择调用方式 🌐             |
     +================================================+
     | 1) 使用 GitHub Raw 调用
     | 2) 使用 jsDelivr 调用
     | 3) 使用 Cloudflare 调用
     | 4) 退出
     +================================================+
     ```
   - 输入选项数字（1、2 或 3）选择调用方式。

3. **进入主菜单**：
   - 选择调用方式后，会进入主菜单：
     ```
     +================================================+
     |                🎯 主菜单 🎯                   |
     +================================================+
     | 1) 系统环境配置
     | 2) 网络环境配置
     | 3) 网络工具配置
     | 4) 简单建站配置
     | 5) Swap 管理
     | 6) 面板管理
     | 7) 退出
     +================================================+
     ```
   - 输入选项数字选择功能模块。

4. **加载子菜单**：
   - 选择功能模块后，脚本会根据用户选择的调用方式动态加载对应的子菜单脚本。
   - 例如，选择“系统环境配置”后，会加载 `system_env.sh` 脚本。

   - 如果需要更新脚本，请重新下载最新版本，或直接通过 GitHub、jsDelivr、Cloudflare 调用最新版本。
